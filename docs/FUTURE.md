# wordSnapV1 候选事项池（FUTURE）

更新时间：2026-03-28

> 本文件用于承接“当前不阻断需求、但有长期价值”的候选事项。
> 进入实现前，仍需按 `docs/TDD_CORE.md` 与 `docs/TDD_PLAN.md` 评估是否采用 TDD。

## 1. 背景说明

- 2026-03-27 手工验证 `M3A-WEB-01` 到 `M3A-WEB-08` 均可快速识别并成功。
- 按当前产品诉求，M3 本轮优化迭代先收口；未执行项转入候选池，后续按实际反馈再启用。

## 2. M3 后续候选（优先级从高到低）

### F1：OCR 候选筛选与决策收敛（原 M3-C）

- 目标：降低噪声词进入查词链路的比例。
- 候选内容：
  - 非英文字符白名单与长度约束；
  - 候选打分/投票选优；
  - 与 `WordNormalizer` 协同去重规则。
- 触发条件：线上/手工反馈出现“误识别高于预期”时启动。

### F2：全屏应用与热键冲突治理（原 M3-D）

- 目标：冲突场景下避免静默失败，给出可执行建议。
- 候选内容：
  - 细分冲突类型（热键注册失败、覆盖层置顶失败、全屏输入拦截）；
  - 设置页增加冲突即时反馈与备用热键建议；
  - 统一冲突提示文案。
- 触发条件：出现“热键无响应/全屏不可触发”的集中反馈时启动。

### F3：预处理策略参数外置与场景化配置（原 M3-B 续项）

- 目标：降低后续调参成本，支持可回退策略。
- 候选内容：
  - 将策略组（阈值、缩放、反色、重试顺序）外置为配置；
  - 增加策略实验记录模板。
- 触发条件：需要针对某类场景快速调参时启动。

### F4：M3 全量基线补测与结项（原 M3-E）

- 目标：形成完整量化结论（30 样本 + 困难子集 + 时延）。
- 候选内容：
  - 补齐剩余样本实测并输出基线 v1；
  - 执行受影响 `unit/integration` 回归；
  - 输出 M3 结项记录。
- 触发条件：准备进入发布强化阶段，或需对外给出量化质量承诺时启动。

### F5：OCR 路径兼容防御测试（来自 P5 待办）

- 目标：补齐 PATH 分隔符与兼容行为回归。
- 候选内容：
  - PATH 分隔符（`;`）边界测试；
  - Windows-first 兼容防御用例。
- 触发条件：路径解析相关回归问题复现时启动。

## 4. 启用规则

- 代码类候选按 `TESTING_GUIDELINES.md` 仅补高价值测试，避免测试膨胀。
